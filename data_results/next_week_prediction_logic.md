## 一、整体架构

```
输入数据
  ├── 个股本周日K线 (stock_kline表)
  ├── 大盘指数本周K线 (stock_kline表)
  ├── 本周预测结果 (this_week_pred, 含资金流向/成交量/财报/板块信号)
  └── 历史K线 (用于价格位置、前周涨跌计算)
      │
      ▼
┌─────────────────────────────────────────┐
│  Step 1: 数据准备                        │
│  - 获取本周K线 (≥3天才预测)              │
│  - 计算大盘本周涨跌幅                    │
│  - 从this_week_pred获取多维信号           │
│  - 计算价格位置(60日高低点)               │
│  - 计算前一周涨跌幅                      │
├─────────────────────────────────────────┤
│  Step 2: 特征提取 (_nw_extract_features) │
│  - this_week_chg: 本周复合涨跌幅         │
│  - market_chg: 大盘本周涨跌幅            │
│  - consec_down/up: 连跌/连涨天数         │
│  - last_day_chg: 最后一天涨跌幅          │
│  - _market_suffix: SH/SZ/BJ             │
│  - _price_pos_60: 价格位置(0~1)          │
│  - _prev_week_chg: 前一周涨跌幅          │
│  - ff_signal/vol_ratio/finance_score等   │
├─────────────────────────────────────────┤
│  Step 3: 规则匹配 (_nw_match_rule)       │
│  - 按优先级遍历_NW_RULES                 │
│  - 命中第一条即停止 (互斥匹配)           │
│  - 未命中 → 返回None (不确定)            │
├─────────────────────────────────────────┤
│  Step 4: 置信度修正                      │
│  4a. 成交量形态修正                      │
│      (_adjust_nw_confidence_by_volume)   │
│  4b. 概念板块强弱势修正                  │
│      (_adjust_nw_confidence_by_board)    │
├─────────────────────────────────────────┤
│  Step 5: 输出                            │
│  - nw_pred_direction: UP/DOWN/None       │
│  - nw_confidence: high/reference/low     │
│  - nw_reason: 可读理由文本               │
└─────────────────────────────────────────┘
```

## 二、个股→大盘指数映射

每只个股根据代码前缀映射到对应的大盘指数，用于计算"大盘本周涨跌幅"：

| 代码前缀 | 市场 | 对应指数 |
|---------|------|---------|
| 000/001/002/003 | 深市主板 | 399001.SZ (深证成指) |
| 300/301 | 创业板 | 399001.SZ (深证成指) |
| 600/601/603/605 | 沪市主板 | 000001.SH (上证指数) |
| 688/689 | 科创板 | 000001.SH (上证指数) |
| 430/830~839/870~873/920 | 北交所 | 899050.SZ (北证50) |

规则中的 `_market_suffix` 取值为 `SH`、`SZ` 或 `BJ`。

## 三、特征提取

函数: `_nw_extract_features(daily_pcts, market_chg, ...)`

### 3.1 本周复合涨跌幅 (this_week_chg)

```python
# 复合收益率计算（非简单加总）
r = 1.0
for p in daily_pcts:
    r *= (1 + p / 100)
this_week_chg = (r - 1) * 100
```

例: 周一+2%, 周二-1%, 周三+3% → (1.02 × 0.99 × 1.03 - 1) × 100 = +4.04%

### 3.2 连涨/连跌天数 (consec_down, consec_up)

从本周最后一天向前计算连续同方向天数，遇到反向即停止：

```
日涨跌: [+1%, -2%, -1%, -3%, +0.5%]
从尾部: +0.5% → consec_up=1, 遇到-3%停止
结果: consec_up=1, consec_down=0

日涨跌: [+1%, +2%, -1%, -2%, -3%]
从尾部: -3%, -2%, -1% → consec_down=3, 遇到+2%停止
结果: consec_down=3, consec_up=0
```

### 3.3 价格位置 (price_pos_60)

当前收盘价在近60个交易日高低点中的相对位置：

```python
# 取历史60日收盘价 + 本周收盘价
all_closes = hist_closes[-60:] + this_week_closes
min_c, max_c = min(all_closes), max(all_closes)
price_pos_60 = (latest_close - min_c) / (max_c - min_c)
# 0 = 60日最低点, 1 = 60日最高点
```

需要至少20条历史K线才计算，否则为None。

### 3.4 前一周涨跌幅 (prev_week_chg)

取本周之前最近5条K线的复合收益率作为前一周涨跌幅。

## 四、规则引擎 (_NW_RULES)

规则按优先级排列，命中第一条即停止（互斥匹配）。未命中任何规则 → 输出"不确定"(None)。

### 4.1 Tier 1 涨信号（准确率≥68%）

#### R1: 大盘深跌 + 个股跌 → 涨

```
条件: 个股本周跌>2% AND 大盘本周跌>3%
预测: 下周涨
回测: 89.6% (6297样本)
逻辑: 大盘深跌后的系统性超跌反弹，最强信号
```

#### R3: 上证 + 大盘跌1~3% + 个股跌>3% + 前周跌 → 涨

```
条件: 市场=SH AND -3%≤大盘<-1% AND 个股跌>3%
      AND 前周跌>2% AND 价格位置<0.8
预测: 下周涨
回测: 68.3% (436样本)
逻辑: 上证连续两周下跌后的均值回归
```

#### R5a: 深证 + 大盘微跌 + 个股跌 + 连跌3天 → 涨

```
条件: 市场=SZ AND -1%≤大盘<0% AND 个股跌>2% AND 连跌≥3天
预测: 下周涨
回测: 89.1% (514样本)
逻辑: 深证大盘仅微跌但个股连续下跌，超跌反弹概率极高
```

#### R5b: 深证 + 大盘微跌 + 个股跌 + 低位 → 涨

```
条件: 市场=SZ AND -1%≤大盘<0% AND 个股跌>2% AND 价格位置<0.2
预测: 下周涨
回测: 78.5% (627样本)
逻辑: 深证个股处于60日低位且本周下跌，底部反弹
```

#### R5c: 深证 + 大盘微跌 + 个股跌>2% → 涨

```
条件: 市场=SZ AND -1%≤大盘<0% AND 个股跌>2%
预测: 下周涨
回测: 75.3% (1802样本)
逻辑: R5a/R5b的宽松版，深证微跌环境下个股跌幅较大即触发
```

#### 资金流向涨信号（实盘触发）

```
规则A: 个股跌>2% + 主力资金流入>0.3 + 量比>1.2 → 涨
规则B: 个股涨>3% + 资金流入>0.2 + 量比>1.0 + 量价相关>0.3 → 涨
说明: 依赖实盘资金流向数据，回测中无法触发
```

### 4.2 Tier 1 跌信号（准确率≥66%）

#### R6c: 深证 + 大盘跌1~3% + 个股涨>2% + 连涨≥3天 → 跌

```
条件: 市场=SZ AND -3%≤大盘<-1% AND 个股涨>2% AND 连涨≥3天
预测: 下周跌
回测: 66.9%
逻辑: 大盘下跌环境中个股逆势连涨，过热回调
```

### 4.3 Tier 2 信号（中等置信，准确率60~68%）

#### R2: 上证 + 大盘跌1~3% + 个股跌>5% + 非高位 → 涨

```
条件: 市场=SH AND -3%≤大盘<-1% AND 个股跌>5% AND 价格位置<0.7
预测: 下周涨
回测: 全样本73.4%(1745) → CV62.0%(326)
说明: 交叉验证差距11.4%，存在过拟合，降为Tier2
```

#### R6a: 深证 + 大盘跌1~3% + 个股涨>5% → 跌

```
条件: 市场=SZ AND -3%≤大盘<-1% AND 个股涨>5%
预测: 下周跌
回测: 全样本73.0%(1292) → CV63.6%(780)
说明: 交叉验证差距9.4%，降为Tier2
```

#### R7: 个股跌>3% + 前期连涨≥3天 + 非高位 → 跌

```
条件: 个股跌>3% AND 连涨≥3天 AND 价格位置<0.6
预测: 下周跌
回测: 71.7% (311样本), CV73.7%
逻辑: 连涨后突然大跌，趋势反转信号（非高位排除正常回调）
```

#### R8: 上证 + 大盘微跌 + 个股涨>2% + 前周跌<-3% → 跌

```
条件: 市场=SH AND -1%≤大盘<0% AND 个股涨>2% AND 前周跌>3%
预测: 下周跌
回测: 全样本74.9%(279) → CV61.5%(156)
说明: CV差距13.4%，保留Tier2但标记不稳定
```

### 4.4 Tier 3 信号（资金流向/财报，实盘触发）

```
资金流出+缩量 → 跌: ff_signal<-0.4 + vol_ratio<0.7 + 个股跌
财报利好+资金流入 → 涨: finance_score>0.5 + ff_signal>0.2
说明: 依赖实盘数据，回测中不触发
```

### 4.5 规则优先级与互斥

```
R1 → R2 → R3 → R5a → R5b → R5c → 资金涨A → 资金涨B
→ R6a → R6c → R7 → R8 → 资金跌 → 财报涨 → None(不确定)
```

注意: R5a/R5b/R5c 是递进关系（R5a最严格，R5c最宽松），由于互斥匹配，满足R5a的样本不会落到R5c。

## 五、置信度修正

规则匹配后，初始置信度由Tier决定：
- Tier 1 → `high`
- Tier 2 → `reference`
- Tier 3 → `reference`

然后依次经过两层修正：

### 5.1 成交量形态修正 (_adjust_nw_confidence_by_volume)

检测5种成交量形态信号，推断量能方向(up/down)，与规则预测方向对比：

| 信号 | 推断方向 | 检测条件 | 信号强度 |
|------|---------|---------|---------|
| 恐慌底 | 涨 | 本周跌 + 放量(>1.3倍) + 低位(<0.25) | 0.1~1.0 |
| 天量阴线 | 跌 | 单日成交量>60日均量3倍 + 收阴 | 0.7 |
| 高位放量跌 | 跌 | 本周跌>1% + 放量(>1.3倍) + 高位(>0.75) | 0.6 |
| 急涨缩量 | 跌 | 前半周涨>2% + 后半周量<前半周60% | 0.5 |
| 价升量缩 | 跌 | 本周涨>0.5% + 量比<0.8 | 0.4 |

修正规则：

```
确认(量能方向=预测方向):
  强信号(strength≥0.5) + reference → high (提升)
  其他 → 保持不变，标注"量能确认"

矛盾(量能方向≠预测方向):
  强矛盾(strength≥0.6):
    high → reference (降级)
    reference → low (降级)
  弱矛盾(strength<0.6):
    high → reference (降级)
    reference/low → 保持不变
```

### 5.2 概念板块强弱势修正 (_adjust_nw_confidence_by_board)

利用个股所属概念板块的本周涨跌情况修正置信度：

- `board_momentum`: 个股所有概念板块本周涨跌幅的均值
- `concept_consensus`: 看涨板块占比 (0~1)

修正规则：

```
预测涨时:
  板块大跌<-3% → 确认(超跌反弹): reference→high, 标注"板块确认↑"
    回测: 91.2% vs 基线82.5% (+8.7pp), 1582样本
  全部看跌(consensus=0) → 确认: 标注"板块共识确认"
    回测: 87.2% vs 基线82.5% (+4.7pp), 1755样本
  板块微跌(-1~0%) → 矛盾(板块没跌够): high→reference, 标注"板块弱矛盾↓"
    回测: 65.7% vs 基线82.5% (-16.7pp), 356样本

预测跌时:
  板块涨>1% → 确认(个股逆势弱): reference→high, 标注"板块确认↑"
    回测: 82.5% vs 基线72.2% (+10.2pp), 114样本
  全部看涨(consensus=1.0) → 确认: 标注"板块共识确认"
    回测: 79.2% vs 基线72.2% (+7.0pp), 72样本
```

### 5.3 修正后置信度分层效果

| 置信度 | 含义 | 准确率范围 |
|--------|------|-----------|
| high | 高置信 | 80~91% |
| reference | 参考级 | 65~75% |
| low | 低置信 | <65% |

板块修正A/B验证（200只×29周）：
- 板块确认: 86.3% (283/328)
- 板块矛盾: 70.2% (33/47)
- 无板块信号: 65.6% (86/131)

## 六、输出字段

函数返回一个dict，关键字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| nw_pred_direction | UP/DOWN/None | 预测方向，None=不确定 |
| nw_confidence | high/reference/low/None | 置信度 |
| nw_strategy | nw_rule_t1/t2/t3/None | 命中的规则层级 |
| nw_reason | str | 可读理由，如"大盘深跌>3%+个股跌→涨; 本周-5.2%; 深证-3.8%; 板块确认" |

## 七、理由文本构成

理由由分号分隔的多个部分组成：

```
[规则名称]; 本周{涨跌幅}%; {指数名}{大盘涨跌}%; [资金流入/流出]; [放量/缩量]; [量能信号]; [板块信号]; [参考信号]
```

示例：
```
深证+大盘微跌+跌+连跌3天→涨; 本周-4.2%; 深证-0.6%; 板块确认↑
大盘深跌>3%+个股跌→涨; 本周-8.1%; 上证-4.3%; 量能确认(恐慌底)
上证+大盘微跌+涨+前周跌→跌; 本周+3.1%; 上证-0.4%; 板块确认↑; 参考信号
```

## 八、数据流与调用链

```
run_batch_weekly_prediction()          # 批量入口
  └── _predict_stock_weekly(code)      # 本周预测 (d3/d4策略)
        ├── _compute_fund_flow_signal()  # 资金流向信号
        ├── _compute_volume_signal()     # 成交量信号
        ├── _compute_finance_signal()    # 财报信号
        └── 输出 this_week_pred
              │
              ▼
  └── _predict_next_week(code, this_week_pred)  # 下周预测

        ├── 计算 price_pos_60, prev_week_chg

        ├── _nw_match_rule()             # 规则匹配

        ├── _adjust_nw_confidence_by_board()   # 板块修正
```

- `stock_kline` 表: 个股日K线（开高低收、涨跌幅、成交量）

- `concept_board_kline` 表: 概念板块日K线
- `stock_fund_flow` 表: 资金流向数据（实盘）
- `stock_finance` 表: 财报数据（实盘）

## 十、不确定(None)的含义

约90%的个股×周组合不会命中任何规则，输出 `nw_pred_direction = None`。

这是设计意图：只在高置信条件下输出预测，避免在随机区间（涨跌各50%）给出误导性信号。

典型不触发场景：
- 大盘涨 + 个股涨（趋势延续，无反转信号）
- 大盘平 + 个股平（无明显方向）
- 个股涨跌幅<2%（波动太小，无法判断）
- 北交所股票（规则主要覆盖沪深）
