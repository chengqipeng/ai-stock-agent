为什么需要正交化？

原始三模型（V11/V20/V30）共享同一个底层维度——"股票跌了多少"。三模型同时看好一只股票，不是三个独立专家都认可，而是同一个判断被重复了三次。

实盘验证：原始三模型同时高置信的72只股票，准确率仅30.6%，比V11单独的46.5%还低。原因是V20/V30的高置信条件把选股范围缩窄到了"跌得最惨的"——这些股票本周平均跌7.89%，是其他股票的1.7倍。

V11 方向判断引擎（下周预测）

V11是方向判断的主引擎，基于本周涨跌幅+大盘环境+交易所，用多层规则匹配预测下周方向。

验证基础：5233只A股 × 29周交叉验证。

规则匹配层级（按优先级）

层级名称说明CV准确率样本占比
层1backbone 骨干V5已验证核心规则（R1/R3/R5/R6/R7）71~90%~65%
层2.5bull 尾日效应大盘涨+尾日跌/冲高回落（仅UP信号）69~79%~3%
层2extension 扩展覆盖V5空白场景的多因子规则70%~22%
层2.5bbull_marginal 边际大盘涨场景的边际规则61~67%~2%
层4fallback 兜底严格条件的通用超跌/过热规则68~72%~8%

核心规则示例

V5_R1: 大盘深跌+个股跌→涨 backbone
条件：this_chg < -2 且 mkt_chg < -3
逻辑：大盘深跌时个股超跌，下周均值回归反弹。CV准确率 89.5%。
V5_R5c: 深证+微跌+跌>2%→涨 backbone
条件：suffix=SZ 且 -1 ≤ mkt_chg < 0 且 this_chg < -2
逻辑：大盘微跌时深证个股超跌反弹。CV准确率 84.8%。
BULL_UP1: 大盘尾日跌>1%+个股跌>2%+低位→涨 bull
条件：mkt_chg ≥ 0 且 mkt_last_day < -1 且 this_chg < -2 且 pos60 < 0.3
逻辑：大盘周涨但尾日跌，个股低位超跌，下周反弹。CV准确率 78.9%。
V5_R6c: 深证+大盘跌+涨+连涨3天→跌 backbone
条件：suffix=SZ 且 -3 ≤ mkt_chg < -1 且 this_chg > 2 且 cu ≥ 3
逻辑：大盘跌但个股逆势连涨，过热回调。CV准确率 73.5%。

V20 资金面确认（正交化后）

V20不独立预测方向，而是回答"资金面是否支持V11的方向判断"。

设计原理

大样本发现（1950样本×39周）：资金大幅流出的超跌股票反弹概率最高（61.8%），资金流入的反而最低（49.3%）。

这符合A股"恐慌性抛售后反弹"的逻辑：大幅流出 = 恐慌抛售已充分释放，卖压衰竭；资金流入 = 可能是主力对倒出货，不是真正抄底。

确认逻辑（V11预测UP时）

high确认 big_net_pct < -5%
恐慌性抛售已充分释放，卖压衰竭，反弹概率高。
medium -5% ≤ big_net_pct < -2%
资金流出中等，部分释放。
deny否决 big_net_pct > -1%
抛压未充分释放（资金流入或微流出），反弹概率低。

确认逻辑（V11预测DOWN时）

high确认 big_net_pct < -5% — 资金持续撤离，确认继续跌
deny否决 big_net_pct > 0% — 资金流入，有人接盘，否定继续跌

与V11的正交性

维度V11看什么V20看什么独立性
价格本周涨跌幅(this_chg)不使用✅
大盘大盘涨跌(mkt_chg)不使用✅
资金不直接使用大单净流入(big_net_pct)✅

V30 结构面确认（正交化后）

V30不独立预测方向，而是回答"价格结构是否支持V11的方向判断"。

设计原理

60日价格位置（price_pos_60）度量股票在近60个交易日高低点中的相对位置。极低位（pos<0.03）的股票反弹概率62.7%，高位（pos>0.55）的只有50.4%。

这个维度独立于V11（V11看本周涨跌幅）和V20（V20看资金流）：同样本周跌5%的两只股票，一只在60日低位一只在60日高位，结构面完全不同。

确认逻辑（V11预测UP时）

high确认 price_pos < 0.20
极低位，下跌空间已充分释放，反弹空间大。
medium 0.20 ≤ price_pos < 0.40
低位，有一定反弹空间。
deny否决 price_pos > 0.55
高位，涨幅空间有限，反弹概率低。

确认逻辑（V11预测DOWN时）

high确认 price_pos > 0.55 — 高位确认有下跌空间
deny否决 price_pos < 0.20 — 极低位否定继续跌

正交共识 三维度交叉验证

V11给出方向判断后，V20和V30分别从资金面和结构面进行独立确认或否决。

共识规则

consensus high
V11 high/reference + V20 high + V30 high — 三个维度全部确认
consensus medium
V11确认 + V20/V30至少两个medium以上 + 无deny
consensus low
V20或V30任一deny — 即使V11高置信也降级

deny否决机制

核心价值：当某个维度明确反对时，即使其他维度都支持，也降低置信度。

场景V11V20V30共识说明
理想反弹UP high恐慌释放极低位high三维度一致
资金否决UP highdeny低位low抛压未释放
结构否决UP high恐慌释放denylow高位无空间
